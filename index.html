GIT Шпаргалка

  • git config --global user.name blabla-додавання імені користувача після установки GIT. Це потрібно для розуміння хто підписує комит
  • git config --global user.email blabal@bla-додавання емейлу користувача після установки GIT. Це потрібно для розуміння хто підписує комит
  • git init-створити порожній GIT
  • git status-переглянути які файли які мають відмінності з файлом індексу і поточним станом
  • git add-підготувати файли до коммітов
  • git commit-створити комит який описує нові зміни
  • git log-відображає зміни кожного коміта
  • git checkout-оновлює файли в робочому дереві відповідно до версії в індексі або вказаному дереві
  • git branch-показує всі гілки і поточну підсвічує зеленим
  • git push-віддати дані в репозиторій
  • git pull-отримати дані в репозиторій
  • git merge-ця команда використовується для злиття гілок
  • git clone-клонує репозиторій в новий каталог

.gitignore-повідомляє git які файли він повинен ігнорувати

Advanced

  • git revert-скасовує новий комміт якщо він був не потрібен
  • git reset-скасовує всі зміни в робочому каталозі
  • git rebase-працює як і merge але не зберігає історію (послідовність) комита. Краще не використовувати в командній роботі, а для локальних гілок
  • git cherry-pick-перетягнути комит з іншої гілки в свою
  • git stash-якщо нам не потрібно робити комміт, але потрібно зберегти поточну версію змін
